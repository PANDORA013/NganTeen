💡 Technical Explanation

The controller methods for orders() and payouts() both check: 

$warung = Auth::user()->warung;
if (!$warung) {
    return redirect()->route('penjual.dashboard')
        ->with('error', 'Warung tidak ditemukan');
}

Previously, User ID 1 had no warung, so this condition was always true, causing infinite redirects back to dashboard. Now with a warung created, the condition is false and the pages load normally.
